# 资源引擎目标与交付边界

## 产品目标

让玩家用一张本地手绘图、上传图、已有资源或一句意象，尽快得到一份能进入游戏 Runtime 的视觉资产。
系统应替用户承担格式、统一风格、尺寸、锚点、模板挂载与低成本动效；不以标准制作流程阻塞
用户表达。

## 唯一成本路由

```text
任意输入：可选的本地画图 / 上传 / 已有资源 / 一句意象 / 模板选择
  -> 本地输入存在时优先：STYLE 1 美化、透明 PNG、revision、绑定、本地库命中
  -> 否则：云端已审核资产精确或近似命中
  -> 确定性变体：裁切、尺寸、改色、风格化、Runtime 变形
  -> 图像编辑：仅缺少少量像素时
  -> 图像生成：确无可用资产时
  -> 验收 -> AssetWorld -> Sprite / UI 模板 / 状态机 -> 可玩 Runtime
```

手绘入口是可跳过的增强能力，不能作为 BuildContract、模板选择、云端复用、确定性变体或模型
调用的前置门槛。任意阶段的失败不得绕过 `Asset Weave` 直接写入 Runtime；应留下可见的
PlaceholderDebt。

## 与语义引擎对齐的结构

| 语义引擎职责 | 资源引擎对应物 |
| --- | --- |
| 语义词典与可写契约 | 资源词典、风格 token、模板角色与输出契约 |
| 意图编译与路由 | `AssetSpec` 与 `AssetResolver` |
| 模块/玩法模板 | UI、美术与动效模板 |
| 编译执行产物 | 透明 PNG、分层 Sprite、绑定与状态机 |
| Runtime 世界投影 | `AssetWorld` |
| 验收、修复、债务 | 规格验证、视觉审查、修复计划、PlaceholderDebt |
| 可追溯记忆 | 本地 revision、资源来源、云端批准记录、变体谱系 |

## 第一阶段：纯本地意象美化

`gamecastle.style-1` 是当前唯一默认风格。输入是用户已经画出的 alpha 轮廓；Runtime 不需要
判断它“是什么”，只需使其不丑、可玩且融入世界：

- 平滑/裁切透明边缘、统一尺寸与底部居中锚点；
- 固定深色粗描边、有限色板、投影与高光；
- 可选通用表情/装饰组件；
- 输出透明 PNG 与不可变 revision；
- 默认 `idle / move / hit / death`，每状态一张资产，Runtime 变形优先。

这一阶段零模型、零云端调用。用户画得无法辨识时，结果仍可以是统一风格的抽象小物，而不
伪造语义理解。

## 本地库与云端库

### 本地库

- 用户私有，浏览器/项目内即时可用；
- 保存原始输入、STYLE 1 输出、revision 与 AssetBinding；
- 用户选择绑定后，项目输出保存内容哈希 PNG 与不含 data URL 的 binding manifest；可玩 release 复制这些导出文件；
- Runtime API 在保存本地 PNG 后必须先走 Asset Weave 的 `local -> validation -> acceptance -> RuntimeLinker` 分支，binding manifest 持久化其 `styleId`、动画 recipe 与状态机；
- 不需审核，不自动上传，不作为其他用户的检索来源。

### 云端库

- 仅接收通过 Acceptance、具有来源与授权信息的资产；
- 提供跨项目精确/近似检索和项目本地缓存；
- Runtime 的云搜索入口只返回已批准资产的安全元数据；物理路径、候选与未审核资源不暴露给前端；
- 精确命中和近似命中的确定性变体都会材料化到项目 `assets/cloud`；导出与离线试玩不引用云仓内部路径；
- 记录批准状态、来源、许可证、标签与变体父子关系；
- 不能覆盖本地用户资产的优先级。

## 验收目标

1. 手绘或上传图可在无网络、无模型时完成美化、裁切、透明 PNG 导出、revision 和绑定。
2. Asset Weave 明确按“本地 -> 云端 -> 确定性 -> 编辑 -> 生成”路由。
3. 所有 Runtime Binding 都携带 `styleId` 与低帧状态机策略。
4. UI 模板、美术模板与 Runtime 都从资源词典读取同一组风格 token。
5. 本地、云端、模型缺失或验收失败均有可测试、可追溯的行为。
